# ResuMax — AI Resume & Cover Letter Optimizer

A multi-model agentic pipeline that takes your existing resume and a target job description, then produces an **ATS-optimized resume** and a **tailored cover letter** in clean, downloadable formats (Word, PDF, plain text, Markdown). The output is deliberately scrubbed of AI/tool fingerprints — anonymous metadata, no smart-quote tells, no telltale preambles — so it reads as human-authored.

Built with FastAPI + a vanilla JS frontend, orchestrating **OpenAI GPT** and **Anthropic Claude** in a five-stage iterative loop.

## What it does

You upload an existing resume (PDF, DOCX, or TXT) and either paste a job description or paste a job-posting URL. The pipeline:

1. **Parses** your resume and (optionally) scrapes the job description from the URL.
2. **Tailors** the resume to the JD via GPT and drafts a matching cover letter.
3. **Self-audits** with an ATS-style scoring pass and **iteratively refines** until the score is ≥ 85 % or three attempts are used.
4. **Reviews** the tailored resume with Claude for recruiter-grade phrasing, action verbs, and structure.
5. **Humanizes** the Claude-edited output with GPT to remove robotic patterns and AI buzzwords.
6. Lets you **download** the final resume and cover letter in Word, PDF, plain text, or Markdown — all with stripped metadata and clean professional typography.

## Features

- **Multi-format input** — PDF (`pypdf`), DOCX (`python-docx`), TXT. Legacy `.doc` is rejected with a friendly hint to save as `.docx`.
- **Job description ingestion** — paste raw text, or paste a URL and the app scrapes + cleans it (BeautifulSoup).
- **Iterative ATS optimization** — runs up to three refinement loops targeting an 85 %+ keyword/structure match.
- **Dual-model orchestration** — GPT-4o-mini for tailoring/refinement/humanizing, Claude Sonnet 4.6 for the recruiter-grade audit pass.
- **Clean, anonymous exports** — `.docx` and `.pdf` outputs have empty author/creator/application metadata, justified body paragraphs, Calibri/Helvetica typography, and no "Generated by AI" tells.
- **Anti-fingerprint text scrubbing** — smart quotes → straight quotes, em/en-dashes → hyphens, emoji stripped, common AI preambles ("Sure, here is…", fenced ```markdown blocks) removed.
- **Local-first secrets** — keys can live in a gitignored `.env` file or in the browser's `localStorage`; never touch a server you don't control.
- **No build step** — vanilla HTML/CSS/JS frontend served as static files by FastAPI.

### Why two models?

- **GPT (4o-mini)** is fast and cheap — well-suited for the heavy lifting in tailoring, ATS scoring, refinement, and final humanization passes.
- **Claude (Sonnet 4.6)** plays the role of a senior recruiter doing a final audit. It catches weak action verbs, vague achievements, and structural issues GPT's own self-critique tends to miss. Using a different model for the audit reduces same-family blind spots.

### Why the humanizer step?

After Claude rewrites for "recruiter-grade" quality, the prose tends to drift toward AI-typical patterns (over-use of "leveraged", "synergized", parallel triplets). The humanizer pass asks GPT to re-edit those patterns out while keeping every achievement intact, giving a final draft that doesn't read as auto-generated.

## Configuration

You have three options for supplying API keys, in order of recommendation:

### Option A — `.env` file (recommended for local dev)

```powershell
Copy-Item .env.example .env
```
Open `.env` and replace the placeholder values with your real keys:
```
OPENAI_API_KEY=sk-proj-...
ANTHROPIC_API_KEY=sk-ant-...
```
`.env` is already listed in `.gitignore`, so it will never be committed.

### Option B — Persistent user environment variables

```powershell
setx OPENAI_API_KEY "sk-proj-..."
setx ANTHROPIC_API_KEY "sk-ant-..."
```
Restart your terminal once after running `setx`. These persist across reboots and across projects.

### Option C — Browser-only

Run the app, open the **Settings** panel in the top-right, paste your keys, and click **Save Configuration**. Keys are stored in the browser's `localStorage` and sent with each request. Convenient, but tied to one browser profile.

> **Precedence:** browser-sent keys win over `.env` win over OS env vars. If you've previously saved keys in the browser and now want `.env` to take over, blank out the fields in the Settings panel and click Save (this clears `localStorage`).

## Usage walkthrough

1. **Upload your existing resume** (PDF, DOCX, or TXT — max 5 MB) via drag-and-drop or the file picker.
2. **Provide the target job description** in one of two ways:
   - **Paste Text:** paste the JD copy directly into the text area.
   - **Job Link:** paste a URL to the job posting and click **Scrape** — the server fetches the page, strips boilerplate, and shows you a preview of what it captured.
3. Click **Launch Optimization Pipeline**. The five timeline steps update in real time as the pipeline progresses (typical run: 30–60 seconds).
4. When complete, the right panel shows five tabs:
   - **Optimized Resume** — the final, humanized output
   - **Cover Letter** — generated from the same JD
   - **ATS Analysis** — final score, missing keywords, formatting/structure suggestions, per-attempt score history
   - **Claude Audit** — Claude's recruiter-grade feedback log
   - **Initial Draft** — the GPT-only first pass, for reference
5. **Download** from the action bar:
   - **Word (.docx)** — clean, anonymous metadata, justified body, Calibri 11 pt
   - **PDF** — server-rendered, no browser headers/footers
   - **Text (.txt)** — ATS-safe plain text with uppercase section headings
   - **Markdown (.md)** — the raw markdown source

## How the "anti-AI fingerprint" works

When you ask an LLM to write a resume and then save it as Word or PDF, three things usually give it away:

1. **Document metadata.** `python-docx` sets `Application` to `"python-docx"` by default; ReportLab sets `Producer` to `"ReportLab"`. Word documents may also carry an `Author` from the system user. The export pipeline scrubs all of these to empty strings (`_scrub_docx_metadata` in `resume_optimizer.py`; explicit empty `title=""`, `author=""`, `creator=""`, `producer=""` arguments to the ReportLab `SimpleDocTemplate`).

2. **Typographic tells.** LLMs tend to emit Unicode curly quotes (`"…"`, `'…'`), em-dashes (`—`), and emoji. Many ATS parsers also choke on smart quotes. The exporter runs `_clean_ai_artifacts()` over the markdown before rendering, normalizing those to ASCII equivalents and stripping emoji entirely.

3. **Telltale preambles.** Models often start responses with "Sure, here is…", "Below is…", or wrap output in ```markdown fences. Those patterns are matched and removed before rendering.

The result is a file that looks like a normal Word document — no metadata pointing to an authoring tool, clean typography, and no leftover model chatter.

## Troubleshooting

| Symptom                                                                   | Fix                                                                                                                                          |
|---------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| `ModuleNotFoundError` on startup                                          | The venv is not active. Run `.\.venv\Scripts\Activate.ps1` (or recreate with `python -m venv .venv` and re-install requirements).            |
| `Error: Missing argument 'APP'.`                                          | You ran bare `uvicorn --reload`. Use `uvicorn main:app --reload`, or `python main.py` (which calls uvicorn with the right target).           |
| `404 not_found_error` about a model name                                  | Anthropic / OpenAI deprecates models over time. The current defaults are `gpt-4o-mini` and `claude-sonnet-4-6` (set in `resume_optimizer.py`). |
| API-key error even after editing `.env`                                  | Browser-stored keys take precedence. Open the Settings panel, blank both fields, click Save, then retry.                                     |
| Long URLs in the on-screen preview overflow the panel                    | The CSS already uses `overflow-wrap: anywhere` on `.markdown-preview *`. Hard-refresh (Ctrl+Shift+R) to clear the cached stylesheet.        |
| `uvicorn main:app --reload` boots but `import resume_optimizer` fails    | Check the indentation/syntax of `resume_optimizer.py`; `python -c "import resume_optimizer"` will surface the exact error.                   |
